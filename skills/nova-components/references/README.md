# Nova Components References

This skill is split into focused reference files so you can load only the UI guidance that matches the task. These references assume Nova's Angular Material MDC, Kendo UI, and `@aya/controls` stack.

## Route By Task

- Use [standalone-and-defaults.md](standalone-and-defaults.md) for standalone component structure, global Material defaults, and base setup
- Use [material-patterns.md](material-patterns.md) for buttons, forms, cards, and chips
- Use [grids-imports-and-state.md](grids-imports-and-state.md) for Kendo Grid patterns, common imports, and choosing local state patterns

## Cross-Skill Boundaries

- Use `nova-angular-standards` for broader Angular coding conventions
- Use `nova-control-flow` for detailed `@if`, `@for`, and `@switch` guidance
- Use `nova-signals` for Angular v17 signal APIs
- Use `nova-state-management` for full NgRx or ComponentStore guidance
- Use `nova-styling` for SCSS and responsive layout details
